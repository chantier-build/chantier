<objective>
Land Wave 1 infrastructure: a single bats test file `core/tests/skill_uniformity.bats` that
encodes D-16 (uniform `harness_adapters` across all shipped skills), D-01 (every skill ships an
executable `run.sh`), and FR-010 (every `PRESSURE.md` has at least two scenarios), plus the
parent fixtures directory `core/tests/fixtures/skills/` that Wave 2 will populate. The test
must `skip` cleanly while `skills/` contains only `.gitkeep`, then turn strict-green the
moment Wave 2 lands the four skill directories — without any further edits.

Purpose: separate cross-skill structural enforcement (one file, three checks) from per-skill
authoring (four parallel plans in Wave 2). This is the architectural responsibility split
recommended by RESEARCH.md §"Architectural Responsibility Map" row 8 (uniformity check) and
Pattern 5 (canonical exemplar). The fixture parent directory is a one-line concession that
lets every Wave-2 plan declare `core/tests/fixtures/skills/<name>/dossier/inputs.yml` as a
new file without needing a `mkdir -p` step in its own task action.

Output: one new bats file, one new `.gitkeep`, both committed; full bats suite remains green
(64 existing tests + 1 new file that emits 3 SKIPs); `chantier --self-test` still green.
</objective>

If the result is empty, calls bats `skip "no skills shipped yet"`. Otherwise: for each directory, opens `<dir>/SKILL.md` and extracts the `harness_adapters` array using the awk extractor shape in RESEARCH §Pattern 5 lines 545–555 (toggles `in_fm` on `^---$` lines, toggles `in_ha` on `^harness_adapters:` line, captures `^[[:space:]]+-[[:space:]]*<value>` lines, strips quotes, sorts, joins with commas). Uses `fail "harness_adapters drift detected: got '<got>', expected '<expected>'"` on mismatch.

    2. `@test "every shipped skill has a PRESSURE.md with at least two scenarios"` — implements FR-010. Same discovery + skip pattern. For each skill dir: asserts `[ -f "<dir>/PRESSURE.md" ]` (`fail "<dir> missing PRESSURE.md (FR-010)"` if absent); then runs `_count=$(grep -cE '^## Scenario [0-9]' "<dir>/PRESSURE.md" || true)` and asserts `[ "$_count" -ge 2 ]` (`fail "<dir> PRESSURE.md has $_count scenarios; need >= 2 (FR-010)"` otherwise).     3. `@test "every shipped skill ships a run.sh per D-01"` — implements D-01. Same discovery + skip. For each dir: `[ -f "<dir>/run.sh" ] || fail "<dir> missing run.sh (D-01: uniform mandate)"`, then `[ -x "<dir>/run.sh" ] || fail "<dir>/run.sh is not executable"`.
