    1. Open each of the four `skills/<name>/SKILL.md` files in your editor:
       - skills/using-git-worktrees/SKILL.md
       - skills/test-driven-development/SKILL.md
       - skills/requesting-code-review/SKILL.md
       - skills/subagent-driven-development/SKILL.md
    2. For each, confirm BY READING (not by greppable structure — the greppable checks already passed):
       (a) The kernel invariants 1–3 are byte-identical across all four files (open a side-by-side diff if needed); the wording in D-06 is reproduced verbatim. Mechanical check: `sha256sum` the three kernel lines from each file's `## Invariants` section — all four should produce the same hash.

    3. For each `skills/<name>/PRESSURE.md`, confirm BY READING:
       (a) The two scenarios are concrete — they describe a specific situation, a specific temptation, a specific required response, a specific Disqualifier. Vague scenarios ("the agent might be tempted to skip the discipline") are not actionable; flag them.
       (b) The Situation paragraphs are believable — they describe pressures an agent would actually encounter (incident on-call, sunk cost from prior work, perceived authority from the user). They are NOT generic ("the agent is busy").
       (c) The Disqualifier subsections each cite an Invariant number AND an `output.json` field that exists in the SKILL.md `outputs_schema` (re-confirm by cross-referencing — the automated check already verified the field name is mentioned; the human check confirms it actually corresponds to a real output field, not a hallucinated one).

       ```

       bats core/tests/
       core/bin/chantier --self-test
       ```
