<objective>
Ship `adapters/claude-code/run-task.sh` — the POSIX-shell harness adapter that stages a Surface-2 dossier inside the operator's pre-created git worktree, dispatches a `claude -p` headless subagent (or the `CHANTIER_CLAUDE_BIN` stub) to read `SKILL.md` and exec `run.sh`, and routes the result through `chantier validate-task` with full D-04 exit-code matrix wiring. Also ship `adapters/claude-code/README.md` for operator usage.

Purpose: Implements FR-008 and locks D-01 through D-16 in code. The adapter is greenfield ~150 lines of glue: every load-bearing capability (`state append`, `validate-task`, gate enforcement, deny-list) already lives in `core/bin/chantier`; this plan only wires them together with `claude -p` dispatch + dossier staging + exit-code bracketing. Resolves all five RESEARCH Open Questions per the recommendations (transcript log on real-claude path, worktree ref in `task.started` refs, defer concurrency lock, defer `chantier task-lookup` subcommand, lax worktree validation).

Output: One executable POSIX shell script (~150 lines) at `adapters/claude-code/run-task.sh`, one English Markdown doc at `adapters/claude-code/README.md`. Both pass `shellcheck --shell=sh` (the README is Markdown so shellcheck does not apply). The adapter is shellcheck-clean and harness-name-allowed only in the directory path comment, error prefix `run-task:`, and dispatch summary string — all sanctioned under D-10.
</objective>
