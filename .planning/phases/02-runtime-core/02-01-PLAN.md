<objective>
Stand up the Wave 0 test and packaging infrastructure for Phase 2 before any binary or schema authoring begins. Install the dev-time toolchain (bats-core, shellcheck), vendor bats-support and bats-assert as git submodules, write `.gitattributes` to prevent CRLF corruption of the future shell binary (RESEARCH Pitfall 3), scaffold five empty `.bats` files so subsequent plans can plug behavior into them, and author four fixture files that downstream tasks (validate-task tests, schema tests) will consume.

Purpose: This is the canonical Wave 0 plan from `02-VALIDATION.md` lines 58–65. Without it, every later plan has nothing to assert against and risks CRLF-bricking the shell binary the first time a non-Unix contributor touches it.

Output: A test directory ready to host real assertions, a vendored helper toolchain that survives `git clone --recurse-submodules`, and a `.gitattributes` guard that protects every later commit of `core/bin/chantier`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → git remote (submodule fetch) | bats-support + bats-assert fetched from github.com/bats-core/ — trusted upstream verified in Task 1 |
| host → Homebrew tap | bats-core + shellcheck installed from core Homebrew formulae — already vetted upstream |
| contributor editor → core/bin/chantier (future commits) | future edits could introduce CRLF on Windows; .gitattributes neutralizes this |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-SC | Tampering | npm/git submodule installs (bats-support, bats-assert) | mitigate | blocking-human checkpoint Task 1 confirms upstream legitimacy before `git submodule add` runs (Task 2). `[ASSUMED]` packages per RESEARCH §Package Legitimacy Audit. |
| T-02-01-CRLF | Tampering | .gitattributes / shell binary | mitigate | Task 3 lays down `core/bin/chantier text eol=lf` BEFORE the binary is authored, so the very first commit of the binary in plan 02-03 is already LF-protected (Pitfall 3). |
| T-02-01-NET | Information Disclosure | submodule fetch | accept | bats-{support,assert} clone is a one-time read-only fetch from a trusted public repo. NFR-004 ("no network at runtime") applies to the shipped binary, not to dev-time setup. |
| T-02-01-FIXTURE | Tampering | test fixtures | mitigate | Fixtures are version-controlled and have no executable bit; they are read-only assets. Any future change to them must pass code review. |
| T-02-01-FS | Tampering | host filesystem | accept | Homebrew install requires sudo only for first-time install on some macOS configurations; this is the user's machine, trust boundary already crossed. |
</threat_model>
