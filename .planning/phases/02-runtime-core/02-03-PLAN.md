## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host shell → core/bin/chantier | the binary is the new trust boundary — all subsequent skill executions invoke it |
| core/bin/chantier → its own source ($0) | self-test introspects $0 for harness leakage and CRLF — read-only file access |
| core/bin/chantier → core/schemas/*.json | self-test calls jq empty on each schema — read-only, parser-bounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-NFR | Spoofing / NFR-001 violation | core/bin/chantier source | mitigate | Self-test (Task 1) and bats test 5 (Task 2) both grep for the 9 harness identifiers, excluding the single literal deny-list grep line. If any other occurrence sneaks in, the test fails and the commit is rejected at CI. |
| T-02-03-CRLF | Tampering | core/bin/chantier source | mitigate | Self-test (Task 1) and bats test 6 (Task 2) both check `file "$0" | grep -q CRLF` returns non-zero. .gitattributes (plan 02-01) is the prevention layer; these checks are the detection layer. |
| T-02-03-INJ | Tampering | --json-errors flag parsing | mitigate | `--json-errors` is a literal-string check, not eval'd. The flag toggles a single integer variable; the value is never expanded into a command. |
| T-02-03-PORT | Spoofing | shebang | mitigate | Test 1 asserts `head -1` equals exactly `#!/bin/sh`. Any drift to bash/zsh shebang fails the test. |
| T-02-03-SC | Tampering | shellcheck-bypass attempts | mitigate | Bats test 8 invokes shellcheck on the binary inside the test suite, so any shellcheck regression is caught at the same gate as test failures. |
| T-02-03-LOCK | DoS | future LOCKDIR contention | accept | LOCKDIR constant declared here is used by plan 02-04's state_append. The mkdir-lock pattern is documented in CONTEXT.md and RESEARCH and is the only portable choice on macOS. Symlink-attack risk on LOCKDIR accepted per RESEARCH §Security Domain row 3 (local-dev trust boundary). |
