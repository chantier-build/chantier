<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| flag values → state_append getopts | user-supplied -e -t -s -m -r values flow into jq via --arg (string-mode, not eval'd) |
| STATE.md → state_show jq pipeline | JSONL body is jq-parsed; malformed lines surfaced as jq errors not silent corruption |
| LOCKDIR mkdir → filesystem | mkdir is atomic; symlink-attack on LOCKDIR accepted per RESEARCH §Security Domain |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-INJ | Tampering | state_append flag values | mitigate | All values flow through `jq --arg` (string-mode binding, NOT executed). Zero `eval` calls in core/bin/chantier (enforced by shellcheck + visual review). RESEARCH §Security Domain row 1 verbatim. |
| T-02-04-REGEX | Tampering | event-name regex (D-09) | mitigate | Two-pass check: cheap shell glob first (PATTERNS-confirmed), authoritative jq `test()` second. Bad event → exit 1, never written to STATE.md. |
| T-02-04-CONCUR | DoS / Tampering | concurrent state_append corruption | mitigate | mkdir-as-mutex with stale-PID detection per RESEARCH Pattern 2. Bats Test 12 stresses 5 concurrent appends and asserts every line parses with `jq empty`. RESEARCH §Security Domain row 5 (TOCTOU) is addressed by the line-atomic `>>` write under the lock. |
| T-02-04-STALE | DoS | orphan LOCKDIR | mitigate | acquire_lock checks `kill -0 $OTHERPID` to detect stale PID. If holder is gone, recover via rm-rf + retry mkdir. Documented in RESEARCH Pattern 2 caveats lines 345–349. |
| T-02-04-PIPE | Tampering | acquire_lock trap loss | mitigate | acquire_lock is called as a top-level statement, NEVER in a pipeline (Pitfall 6). Verified by grep: `acquire_lock |` returns no matches. |
| T-02-04-EMPTY | InfoDisclosure / Repudiation | empty actor field | mitigate | `git config user.name || printf 'unknown'` fallback per Pitfall 9. Test 13 enforces. |
| T-02-04-COLLAPSE | Repudiation | BSD column field collapse | mitigate | Null/empty → `-` substitution in jq before `column -t` (Pitfall 2). Test 2 enforces. |
| T-02-04-LOCK-RO | DoS | state_show blocking writer | mitigate | state_show never acquires the lock — read-only awk+jq snapshot. Test 6 enforces. |
| T-02-04-SYM | Tampering | symlink attack on LOCKDIR | accept | Per RESEARCH §Security Domain row 3 — LOCKDIR lives under `.planning/` which the user already trusts; if `.planning/` is hostile the whole framework is compromised. Acceptable risk for a local-dev tool. |
</threat_model>
